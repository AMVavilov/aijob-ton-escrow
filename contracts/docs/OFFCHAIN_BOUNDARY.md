# On-Chain / Off-Chain Boundary

## Что делает контракт on-chain

- принимает депозит от заказчика
- хранит средства сделки
- хранит адреса сторон и адрес площадки
- хранит `rewardAmount`, `feeAmount`, `totalAmount`
- хранит дедлайны
- переводит сделку между состояниями
- выполняет итоговый payout, split или refund
- хранит settlement metadata:
  - `fundedAmount`
  - `customerPayout`
  - `executorPayout`
  - `feePayout`
  - `resolutionType`
  - `finalizedAt`

## Что НЕ делает контракт

- не читает чат
- не анализирует доказательства
- не принимает AI-решение сам по себе
- не вычисляет смысл спора
- не работает с задачами `AI Job` напрямую
- не хранит большие тексты, историю переписки или вложения
- не вычисляет fair split самостоятельно

## Что делает off-chain слой

- создаёт сделку на основе данных из `AI Job`
- подготавливает deploy параметров контракта
- показывает сторонам статусы сделки
- отслеживает on-chain события
- собирает переписку и артефакты задачи
- даёт боту контекст для первичного разбора спора
- готовит summary для площадки
- вызывает on-chain финальное действие от имени площадки:
  - `platformRelease`
  - `platformRefund`
  - `platformResolveSplit`

## Роли

### Заказчик

- создаёт сделку
- пополняет контракт
- подтверждает выполнение
- может открыть спор

### Исполнитель

- принимает условия сделки off-chain
- выполняет задачу
- отмечает результат как доставленный
- может открыть спор

### Бот

- отслеживает переписку
- фиксирует факты
- собирает timeline
- предлагает первичное решение
- может предложить split по спору, но не двигает деньги сам по правилам `v1`

### Площадка

- видит summary по спору
- принимает финальное решение
- вызывает `platformRelease`, `platformRefund` или `platformResolveSplit`

## Почему это важно

Такое разделение позволяет:

- держать контракт простым и проверяемым
- не тащить AI-логику on-chain
- быстро выпускать `v1`
- в следующих версиях заменить арбитражную механику без ломки старых сделок
